Overview & vision
Problem
When parents (especially Arab/bilingual ones) hand the keyboard to a toddler, existing toys like TinyFingers show only English animations, have no Arabic letter recognition, no bilingual UI, and no culturally relevant themes. Kids see random characters, not their own alphabet.
Solution
ArabFingers is a fullscreen, bilingual keyboard-smash experience. It detects which language the key belongs to, shows the large Arabic or English letter with its name spoken aloud, and fills the screen with animated 3D floating toys, animals, and emojis. The UI is fully localized in Arabic (RTL) and English (LTR) — switchable from a parent panel.
Differentiators vs TinyFingers
Arabic alphabet detection 3D floating objects (Three.js) Bilingual letter display Letter name + sound (TTS) RTL / LTR UI switching Arabic cultural themes Open-source self-hostable

Core features
⌨️
Key capture & display
Any keypress shows the character large on screen. Detects Arabic vs Latin. Shows both scripts side by side for the same key.
🌐
Bilingual letter panel
Shows Arabic letter + English equivalent + the letter name in both languages. E.g. press "A" → shows أ / A / "ألف".
🔊
Text-to-Speech
Web Speech API reads the letter name aloud in the active language (Arabic voice for Arabic letters, English voice for English).
🧸
3D floating objects
React Three Fiber canvas floats 3D toys, animals, stars, and emojis that bounce and spin behind the letter display.
🎨
Themes
Space 🚀, Underwater 🐠, Desert 🌵, Jungle 🦁, Ramadan ✨ — each with matching 3D objects and color palettes.
📱
Touch + mouse + keyboard
Tap anywhere on a touch screen, click, or smash the keyboard — all trigger the same experience. Works on tablets.
🔒
Fullscreen lock
App goes fullscreen immediately and traps navigation keys (F5, Alt+F4, Win key softblock). Parent unlock via secret gesture.
🎉
Confetti milestones
Every 10 keys smashed triggers a confetti explosion + encouraging message in the active language.
4
Localization requirements
Arabic ↔ English mapping
Full 28-letter Arabic alphabet mapped to nearest Latin phonetic equivalent. Arabic keyboard layout (when OS is in AR mode) detected via Unicode range U+0600–U+06FF. Display both scripts simultaneously on each keypress.
أ — Alef ب — Ba ت — Ta ث — Tha ج — Jeem
UI language switching
All UI text (counters, milestones, parent panel, settings) available in Arabic (RTL, dir="rtl") and English (LTR). Stored in /locales/ar.json and /locales/en.json. Active locale stored in localStorage. next-intl handles routing and translation.
Font requirements
Arabic: IBM Plex Arabic or Noto Naskh Arabic (Google Fonts). English display: Fredoka or Nunito (rounded, child-friendly). Both loaded via next/font.

Tech stack
Next.js 15+ (App Router)
Framework. Use App Router with a single root page.tsx that renders the fullscreen app. No SSR needed for the main canvas — use "use client" directive. next-intl for i18n routing (/en, /ar).
React Three Fiber + @react-three/drei
3D canvas for floating objects. Use <Canvas> as a background layer. Drei helpers: Float, MeshWobbleMaterial, Sphere, Box, Torus, OrbitControls (disabled in production). drei/Text for 3D letter rendering.
Framer Motion
2D overlay animations. Letter pop animation (scale + fade), confetti, milestone messages, emoji bursts on keypress. Use AnimatePresence for letter transitions.
Zustand
Lightweight global state for: current key, key count, active theme, active locale, parent panel open state, sound setting. No Redux overhead needed.
next-intl
Internationalization. Translation files at /messages/ar.json and /messages/en.json. RTL detection via locale — apply dir="rtl" to html element when locale is "ar".
Howler.js
Sound effects — key smash sound, milestone chime, confetti pop. Pre-load audio sprites for instant playback. Web Speech API (native browser) for letter name TTS.
Tailwind CSS v4
Utility styling. Custom theme tokens for each visual theme (space, desert, etc.). Dark mode = always on for the main canvas (space feel). RTL utilities via tailwindcss-rtl plugin.
canvas-confetti
Milestone confetti explosions. Triggered at 10, 25, 50, 100 key smashes. Bilingual celebration message overlaid via Framer Motion.
6
3D & animation spec
3D floating objects (R3F)
Background Three.js canvas renders 15–25 floating 3D objects based on the active theme:
Space: 🚀 rockets, ⭐ stars, 🪐 planets Desert: 🌵 cacti, 🐪 camels, 🌙 moons Jungle: 🦁 lions, 🌴 palms, 🐸 frogs Underwater: 🐠 fish, 🐙 octopi, 🪸 coral
Each object uses drei's <Float> for gentle floating, random rotation speed, and MeshWobbleMaterial for jelly-like movement. On keypress, nearest object "reacts" with a bounce (scale spring).
Letter display animation (Framer Motion)
On each keypress: previous letter exits with scale(0) + opacity(0) in 150ms. New letter enters with spring scale from 0.5→1.2→1.0, opacity 0→1. Letter card shows Arabic character (large, ~30vw) + English character (medium, ~10vw) side by side. Both letter names shown below in respective fonts.
Emoji burst
On each keypress, 3–5 random emojis burst from the tap/click point using Framer Motion (radial spread + fade out over 600ms). Each emoji has random velocity vector and rotation.

Parent panel
Access method
Hold top-left corner for 2.5 seconds, OR type "والدين" (Arabic) or "parent" (English) on keyboard. Panel slides in as a modal overlay. Simple PIN lock optional (4-digit, stored in localStorage).
Controls
🔊 Sound on/off 🌍 Language: AR / EN / Both 🎨 Theme picker 🔢 Show key counter 🐢 Reduce motion ⤢ Exit fullscreen 🔤 Letter mode: AR only / EN only / Both 🔊 TTS voice speed
